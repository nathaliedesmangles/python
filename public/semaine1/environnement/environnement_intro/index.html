L'environnement de travail et Introduction à Python

Objectifs d’apprentissage

À la fin de cette leçon, devez être en mesure de :

  • Utiliser VS Code et Jupyter Notebook
  • Créer et exécuter une cellule de code
  • Définir des variables
  • Manipuler des types de base : int, float, str, bool
  • Gérer les entrées (saisies au clavier) et les sorties (affichage) d’un programme.

Présentation des outils de programmation

1. Visual Studio Code (VS Code)

Visual Studio Code est un éditeur de code moderne, simple et très utilisé pour programmer avec Python. Il vous permet aussi de bien organiser vos projets grâce à une gestion intuitive des dossiers et fichiers.

Ce que vous pouvez faire avec VS Code :

  • Écrire et exécuter du code Python
  • Ouvrir et modifier des notebooks Jupyter (.ipynb)
  • Voir tous les fichiers de votre projet dans un seul endroit
  • Lire les erreurs et déboguer plus facilement

2. Jupyter Notebook

Jupyter Notebook vous permet d’écrire du texte et du code Python dans un même document. C’est un outil idéal pour expérimenter, visualiser des données et commenter vos résultats.

Pourquoi l’utiliser ?

  • On peut écrire du code Python et l’exécuter cellule par cellule.
  • On peut insérer des explications ou des titres entre les blocs de code.
  • Très pratique pour les calculs scientifiques, les graphiques et les analyses de données.

Comment l’utiliser dans VS Code

  1. À partir de Moodle (Section Semaine 1), téléchargez le fichier introduction.ipynb.

Par défaut, le fichier se trouvera dans le dossier Téléchargement.

  1. Via l’Explorateur de fichiers, déplacer (Glisser-Déposer) le fichier introduction.ipynb du dossier Téléchargement vers le dossier Exercices

  1. Revenez dans VS Code, assurez-vous de voir le fichier introduction_pyton.ipynb dans le dossier exercices.

  1. Une fois ouvert, à droite, vous verrez des cellules :

    • Certaines celules servent à écrire du texte explicatif (format Markdown)

    • D’autres servent à écrire du code Python

    • Cliquez sur ▶ pour exécuter une cellule

Bonnes pratiques

  • Utilisez un dossier par thème ou par semaine
  • Nommez vos fichiers clairement : calcul_masse.ipynb, analyse_temp.ipynb

La sauvegarde automatique ou sauvegardez régulièrement

Si un rond blanc est présent à droit du nom du fichier dans l’onglet, c’est parce qu’il n’est PAS sauvegardé (comme dans l’exemple ci-dessous).

Pour activer la sauvegarde automatique, cliquez l’onglet Fichier, puis cliquez sur Enregistrement automatique.

Vous devriez maintenant voir un crochet à gauche de cette option.

  • Pour ce fichier, VS Code détectera les modifications et fera la sauvegarde automatiquement, mais vous pouvez à tout moment appuyer sur Ctrl + S pour forcer la sauvegarde

Il devrait maintenant avoir un X à gauche du nom du fichier dans l’onglet du haut. Ce qui signifie que le fichier est sauvegardé.

Fermez les notebooks proprement

  • En haut à droite du nom d’un notebook, cliquez sur X pour fermer le fichier

Les commentaires

  • Les commentaires commencent par le symbole # (dièse ou hashtag).
  • Les commentaires ne sont pas “vus” par Pyhon, donc ils peuvent sevir à “désactiver” temporairement des lignes de code, lors de tests.
  • Les commentaires sont utilisés pour expliquer du code

Un commentaire doit expliquer le pourquoi, pas le comment, que l’on peut souvent lire dans le code.

Exemple utile :

# Conversion de la masse en grammes vers kilogrammes
masse_kg = masse_g / 1000

Commentaires inutiles :

masse = 3  # On met 3 dans masse

À retenir :

  • Ne commentez pas l’évidence
  • Mettez un commentaire en haut du programme pour résumer son objectif.
  • Utilisez des blocs de commentaires pour expliquer des parties complexes.

À retenir
  • Python est sensible à la casse : Nom ≠ nom
  • Les noms de variables ne doivent pas commencer par un chiffre, ni avoir d’espace, ni de lettre accetuée.
  • Utilisez des noms clairs et significatifs (ex : masse, volume, vitesse)
  • Posez vous toujours la question: “Est-ce que ma variable est du bon type ?” que ce soit lors de la création d’une variable ou une variable qui contien le résultat d’une formule.
